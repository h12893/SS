Azure Cognitive Search（Azure AI Search）の SKU（価格レベル）は、性能・ストレージ容量・利用できる機能が大きく異なります。PoC は Free、本番は Basic 以上、RAG では Standard（S1〜S3）が一般的です。  
以下に 各 SKU の違いを用途別に整理 します。

🔎 Azure Cognitive Search の SKU 一覧と違い（要点）
Azure 公式情報より、SKU は以下の 4 系統に分類されます：

Free（無料）

Basic

Standard（S1 / S2 / S3 / S3 HD）

Storage Optimized（L1 / L2）  

🧭 SKU の違いを一言でまとめると
SKU	特徴	向いている用途
Free	50MB 制限、共有環境、機能制限多い	チュートリアル、PoC
Basic	小規模本番向け、低コスト	小規模 FAQ、軽量検索
Standard S1	一般的な本番向け、RAG で最も利用される	社内検索、RAG
Standard S2	S1 の 4〜5倍性能	大規模検索、重い RAG
Standard S3	さらに高性能	高負荷検索、複雑検索
Standard S3 HD	小さなインデックスを大量に持つ用途	マルチテナント SaaS
Storage Optimized L1/L2	TB 級ストレージ、低コスト/GB、遅延は大きめ	大規模文書検索、アーカイブ検索

🧭 実際の運用コストの考え方
1. 本番運用は最低 3 レプリカが必要（SLA 要件）
→ 実際の月額は 上記価格 × 3 が基本。
例：S1 の本番構成 → 約 $750 / 月

2. パーティション数を増やすと SU が増える
大規模データ（数百万〜数千万ドキュメント）では
レプリカ × パーティション = 6〜12 SU になることも。

3. AI エンリッチメントは別料金
Cognitive Services の利用分が追加で発生。

🎯 結論（最重要ポイント）
Free SKU では “ベクトル検索（semantic search / RAG の核）” が使えません。
つまり：

意味検索ができない

ユーザのスキル → 関連資格を自動で見つける

スキルチェックリストとのマッチング

RAG の Retrieval（検索）部分

これらが ほぼ不可能 になります。

🧩 Free SKU の主な制限（初心者向けに簡潔に）

ベクトル検索（RAG の必須機能）	❌ 使えない	RAG が成立しない
Semantic search（意味検索）	❌ 使えない	スキルと資格のマッチング不可
インデックス数	3 個まで	資格＋スキルでギリギリ

🧭 では、どの SKU が必要なのか？
Standard（S1）以上が必須
理由：

ベクトル検索が使える

Semantic search が使える

RAG の Retrieval が構築できる

インデックス数・スループットが十分

💡 コストを抑えたい場合の現実的な選択肢
✔ 選択肢 1：開発は Free、実用は Standard
Free でインデックス作成やデータ投入の練習

Standard に切り替えて RAG を構築
→ 最も現実的

✔ 選択肢 2：Standard を短期間だけ使う
開発期間だけ Standard

完成後に停止
→ 月額課金なので、使う期間を短くすればコストは抑えられる

✔ 選択肢 3：Azure OpenAI の Embedding + 自前ベクトルDB（FAISS）
Cognitive Search を使わずにローカルでベクトル検索

ただし Azure 上で動かすなら結局 VM 代がかかる
→ コストは下がらないことが多い
