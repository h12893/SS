# バージョン管理に関して

バージョン管理はセマンティックバージョニングの形式を基本とするが、本件特有の以下のルールで管理する。
- Ver以降に3つの数字をピリオドで繋げる（初期の記述ではピリオドが抜けている場合有り）
- 3つの数字はそれぞれ「メジャー」「マイナー」「パッチ」で、本来の管理範囲と、本件での管理範囲は以下の通り
    - メジャー（Major）
        - 本来の管理範囲：互換性がなくなるほどの大きな変更の管理（ex.画面の使い勝手を全く違うものに変えた、古いデータが使えなくなるような根本的なシステム変更をした、など）
        - 今回の管理範囲：出力の内容や範囲が変わる機能追加や根本的なシステムの構成変更
    - マイナー（Minor）
        - 本来の管理範囲：今までの機能はそのままに、新しい機能を追加したときに数字を上げる（ex.お気に入り機能を新規追加した、ダークモードに対応した、など）
        - 追加の管理範囲：精度や処理性能の向上のための機能追加や構成変更
    - パッチ（Patch）
        - 本来の管理範囲：バグ（不具合）の修正をした時に数字を上げる（「特定のボタンを押すとアプリが落ちる不具合」を直したとき、誤字脱字を修正したときなど）
        - 追加の管理範囲：内部の処理やコードの整理、出力結果や内容の本質に影響しない修正（誤字脱字、UI、ファイルの命名など）  

運用ルール
- マイナーを上げるとパッチを0にリセット
- メジャーを上げるとマイナーとパッチを0にリセット

#### プロンプト形式
```
あなたはデータサイエンス協会のスキルチェックリストに基づいて、
ユーザが満たしているスキルを判定するアセッサーです。

以下のコンテキストには、スキル項目（base/value/DS/DE/fusion）が含まれています。
ユーザの質問内容から、ユーザが「満たしている」と判断できるスキルを抽出し、
必ず次の JSON 形式で返してください。

出力形式（厳守）：

{{
  "base": [
    {{"skill": "スキル名", "level": 数値}},
    ...
  ],
  "value": [...],
  "DS": [...],
  "DE": [...],
  "fusion": [...]
}}

制約：
- JSON 以外の文章は一切出力しない
- もしコンテキストに該当スキルが見つからない場合でも、ユーザの記述内容から明確に判断できる場合はスキルを含めてよい。
- レベルは該当スキルの最大レベルを返す
- 曖昧な場合は含めない
- 絵文字は禁止
- 日本語で返す
- 不要な記号（\\n や *** など）は使わない

# コンテキスト
{context}

# ユーザ質問
{query}

# JSON 出力
```

#### 出力形式
##### LLM回答生成
- max_tokens
    - 1.00：16384
##### LLM回答からJSON部分のみ抽出
##### レーダーチャート用に抽出JSONのレベル部分をカテゴリ別に集計
##### 抽出JSONのファイル出力
- 出力先・ファイル名：`./output/skills/skills_{skillsフォルダ内の既存ファイル数}.json`
##### レーダーチャート用集計結果のファイル出力
- 出力先・ファイル名：`./output/radar/radar_{radarフォルダ内の既存ファイル数}.json`
##### API
- /rag：生成された回答（上記の抽出JSONとレーダーチャート用集計結果）の表示
- /health：システムの状態を表示
##### レーダーチャートのpngファイル出力
- 出力先・ファイル名：`./output/radarchart/radarchart_{radarchartフォルダ内の既存ファイル数}.png`
