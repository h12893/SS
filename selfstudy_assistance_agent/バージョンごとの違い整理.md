## 構成

### Embeddingモデル
- モデル：all-MiniLM-L6-v2

### ベクトルDB
- モデル：chromaDB
- name："rag_collection"
- metadata：{"hnsw:space": "cosine"}

### ローカルLLM
- モデル：Llama-3-8B-Instruct-Q4_K_M  
- n_ctx：4096  
- n_threads：4

### JSONデータ
使用するJSONデータ一覧
- 資格一覧_ver1.00.json（社内の旧報奨金制度対象の資格 + 関連、類似資格10件）

### 出力
- n_results：5
- スコアフィルタリング：無し

#### プロンプト形式
```
あなたはデータサイエンス分野のキャリア支援エージェントです。
以下の情報を参考に、ユーザの質問に答えてください。
- 出力は必ず **日本語** にする
- 絵文字（😊 など）は一切使用しない
- 不要な記号（\\n や *** など）は使わず、自然な文章として整形する
- 箇条書きは「・」を使う

# コンテキスト
{context}

# ユーザ質問
{query}

# 回答
```

#### 出力形式
##### API
- /rag：生成された回答の表示
- /health：システムの状態を表示

# PoC ver1.

出力に`スキルチェックシート基準での能力のレーダーチャートによる可視化`を追加。

## 構成

### Embeddingモデル
- モデル：all-MiniLM-L6-v2

### ベクトルDB
- モデル：chromaDB
- name
    - 1.00："rag_collection_v1.00"
- metadata：{"hnsw:space": "cosine"}

### ローカルLLM
- モデル：Llama-3-8B-Instruct-Q4_K_M  
- n_ctx
    - 1.00：16384  
- n_threads：12

### JSONデータ
使用するJSONデータ一覧
- 資格一覧_ver1.00.json（社内の旧報奨金制度対象の資格 + 関連、類似資格10件）
- skillcheck_ver6.00.json（データサイエンティスト協会のスキルチェックリストVer6.00）

### 出力
- n_results：50
- スコアフィルタリング
    - 1.00：cosin距離0.8未満

#### プロンプト形式
```
あなたはデータサイエンス協会のスキルチェックリストに基づいて、
ユーザが満たしているスキルを判定するアセッサーです。

以下のコンテキストには、スキル項目（base/value/DS/DE/fusion）が含まれています。
ユーザの質問内容から、ユーザが「満たしている」と判断できるスキルを抽出し、
必ず次の JSON 形式で返してください。

出力形式（厳守）：

{{
  "base": [
    {{"skill": "スキル名", "level": 数値}},
    ...
  ],
  "value": [...],
  "DS": [...],
  "DE": [...],
  "fusion": [...]
}}

制約：
- JSON 以外の文章は一切出力しない
- もしコンテキストに該当スキルが見つからない場合でも、ユーザの記述内容から明確に判断できる場合はスキルを含めてよい。
- レベルは該当スキルの最大レベルを返す
- 曖昧な場合は含めない
- 絵文字は禁止
- 日本語で返す
- 不要な記号（\\n や *** など）は使わない

# コンテキスト
{context}

# ユーザ質問
{query}

# JSON 出力
```

#### 出力形式
##### LLM回答生成
- max_tokens
    - 1.00：16384
##### LLM回答からJSON部分のみ抽出
##### レーダーチャート用に抽出JSONのレベル部分をカテゴリ別に集計
##### 抽出JSONのファイル出力
- 出力先・ファイル名：`./output/skills/skills_{skillsフォルダ内の既存ファイル数}.json`
##### レーダーチャート用集計結果のファイル出力
- 出力先・ファイル名：`./output/radar/radar_{radarフォルダ内の既存ファイル数}.json`
##### API
- /rag：生成された回答（上記の抽出JSONとレーダーチャート用集計結果）の表示
- /health：システムの状態を表示
##### レーダーチャートのpngファイル出力
- 出力先・ファイル名：`./output/radarchart/radarchart_{radarchartフォルダ内の既存ファイル数}.png`
