# population_curated

## prefecture_code 抽出
- 全角数字 → 半角数字変換
- SUBSTRING(prefecture,1,2) を変換
- 特別扱い:
  - 全国 → 0
  - 外国 → 48
  - 不詳 → 49

## MERGE ロジック
- prefecture_code + year をキーに更新/追加
- 数値項目は TRY_CAST

# prefecture_metadata_codes

## prefecture 名称クレンジング
- 都道府県・政令市 → 先頭2桁コード削除
- 全国・外国・不詳 → コード削除しない
- 全角スペース・半角スペース・タブ削除

## region_code と region_type 付与
- 0: 全国
- 1: 都道府県
- 2: 政令市、23区
- 3: 外国
- 4: 不詳

# active_job_openings_curated

## prefecture の空白埋め
- prefectureカラムが空白の場合、直近の値で埋める（都道府県と業種に関して縦持ちなので、同一都道府県で別業種のレコードでは最初の1回以外は都道府県が空白になっている）
- 直近の値は、**row_numの降順にソートした場合の該当レコードのrow_numより小さいrow_numの範囲で、prefectureが空白でない最大のrow_numのレコードのprefectureの値**で定義

## prefecture_code 抽出
- 全角数字 → 半角数字変換
- SUBSTRING(prefecture,1,2) を変換
- 特別扱い:
  - 全国 → 0
  - 外国 → 48
  - 不詳 → 49

## job_code 抽出
- 元データの表示俊に1~86でコード値を振る
- 業種は全体、中分類（頭文字がアルファベットのものや分類不能の業種）、小分類（各中分類内の詳細な分類）が混在するが、job_codeはこれらを区別せずに付与する

## UNPIVOT処理でlanding_rawテーブルを縦持ちのlandingテーブルに変換
- rawデータでは都道府県と業種に関しては縦持ち、年度に関しては横持
- UNPIVOTUNPIVOT処理で年度に関しても縦持ちにして、完全な縦持ち（各レコードが特定の都道府県コード、年度、業種コードで一意で求人数を示すテーブル構造）のlandingテーブルに変換

## PIVOT諸例でlandingテーブルを業種コードに関して横持のcuratedテーブルに変換
- 業種別のカラムをjog_XX（XXは各職業コード）で横持に展開

## MERGE ロジック
- prefecture_code + year をキーに更新/追加
- 数値項目は TRY_CAST
