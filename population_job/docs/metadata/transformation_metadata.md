# active_job_openings_curated

### prefecture の空白埋め
- prefectureカラムが空白の場合、直近の値で埋める（都道府県と業種に関して縦持ちなので、同一都道府県で別業種のレコードでは最初の1回以外は都道府県が空白になっている）
- 直近の値は、**row_numの降順にソートした場合の該当レコードのrow_numより小さいrow_numの範囲で、prefectureが空白でない最大のrow_numのレコードのprefectureの値**で定義

### prefecture_code 抽出
- 全角数字 → 半角数字変換
- SUBSTRING(prefecture,1,2) を変換
- 特別扱い:
  - 全国 → 0
  - 外国 → 48
  - 不詳 → 49

### job_code 抽出
- 元データの表示順に1~86でコード値を振る
- 業種は全体、中分類（頭文字がアルファベットのものや分類不能の業種）、小分類（各中分類内の詳細な分類）が混在するが、job_codeはこれらを区別せずに付与する

### UNPIVOT処理でlanding_rawテーブルを縦持ちのlandingテーブルに変換
- rawデータでは都道府県と業種に関しては縦持ち、年度に関しては横持
- UNPIVOTUNPIVOT処理で年度に関しても縦持ちにして、完全な縦持ち（各レコードが特定の都道府県コード、年度、業種コードで一意で求人数を示すテーブル構造）のlandingテーブルに変換

### PIVOT諸例でlandingテーブルを業種コードに関して横持のcuratedテーブルに変換
- 業種別のカラムをjog_XX（XXは各職業コード）で横持に展開

### MERGE ロジック
- prefecture_code + year をキーに更新/追加
- 数値項目は TRY_CAST

# active_job_seekings_curated

active_job_openings_curatedと同様の処理に加えて以下の処理を実行。  
ただし、「UNPIVOT処理でlanding_rawテーブルを縦持ちのlandingテーブルに変換」「PIVOT諸例でlandingテーブルを業種コードに関して横持のcuratedテーブルに変換」「MERGE ロジック」以外は、データレイクへの格納前に手作業で実施。

### 性別に関して縦持ち
- 元データは「性計」「男」「女」でカラムが分かれているので、これらを3つのテーブルに分割しsexカラムを付与
- 分割後のテーブルをUNION ALLで縦に結合することで、性別に関して縦持ちのlandingテーブルに変換

### sex_code 付与
- 性別の分類に0~2でコード値を振る

### generation_code 付与
- 年代の分類に0~11でコード値を振る

# laborforce_generation_industry_employ_curated

### generation_code 付与
- 年代の分類に0~11でコード値を振る

### year の数値化
- '年'を削除して整数型に変換

### job_X の文字列をNULLに変換
- 以下の記号が入ったレコードはNULLに変換（理由は原文のまま）

| 記号 | 理由 |
|-----|-----|
| * | 分母が小さいため比率を計算していない。|
| *** | 該当数値がないもの　又は　集計していないもの(2002年２月以降)。　なお、2002年１月以前については http://www.stat.go.jp/data/roudou/chuui.htm 参照 |
| … | 東日本大震災の影響により、岩手県、宮城県及び福島県において、調査の実施が困難となりました。このため、平成23年3月分から8月分までの期間を含む全国及び東北地域の結果については掲載していません。 |

### 全てのレコード、カラムの空白削除
- LTRIMとRTRIMでセル単位の両サイドの空白を削除
