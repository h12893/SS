ARMテンプレートや個別JSONによるADFの復元方法をまとめる。

# ARMテンプレートによる復元

ARMテンプレートは**ADFの完全スナップショット**なので、「取得時点のADF全体」への復元が可能。

## ARMテンプレートによる復元ケース

- ADF全体を過去の状態に戻したい場合
- 大規模な破損が発生した場合
- 別環境へ移行したい場合
- 多数のパイプライン・データセットが壊れた場合

## 復元手順

1. Azure Portalを開く
2. 「リソースの作成」→「テンプレートのデプロイ」
3. 「独自のテンプレートを作成する」
4. 該当する日付の ARMTemplateForFactory.json ファイルを貼り付け
5. パラメータ画面で ARMTemplateParametersForFactory.json の内容を入力
6. デプロイを実行

## 注意点

- 既存リソースは上書きされる
- リンクサービスの接続情報は環境依存
- Integration Runtimeの設定には環境債が影響する（自ホストIRは別途設定が必要）

# 個別JSONからの復元

個別JSONでは「ADFの一部だけ」の復元が可能。

## 個別JSONによる復元ケース

- 特定のパイプラインだけ戻したい場合
- 軽微な修正を元に戻したい場合
- GitHubの差分を見ながら戻したい場合

## 復元手順

1. ADF Studioを開く
2. 対象パイプラインを開く
3. 右上の「コード（</>）」をクリック
4. JSON全体を削除して、バックアップしたJSONを貼り付け
5. 保存

データセット、リンクサービス、トリガーそれぞれで上記の手順を実行する。

# 復元時に確認すべきポイント

① リンクサービスの接続情報
- Key Vaultを使っていればARMテンプレートでも安全
- 接続文字列を直接入れている場合は、復元後に再設定が必要

➁ Integration Runtime
- 字ホストIRはARMテンプレートでは復元されない
- Azure IRはARMテンプレートで復元される

➂ トリガーの状態
- ARMテンプレートによる復元後はトリガーが停止状態になるので、必要に応じて手動で「開始」する必要がある

④ パイプラインの依存関係
- 個別JSONで戻す場合、依存するデータセットやリンクサービスも確認する
